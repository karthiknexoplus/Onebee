Vehicle Presence API
Endpoint

POST http://192.168.1.3:5000/api/vehicle/presence

Request Format
{
    "lane_id": "L1",
    "device_id": "D1",
    "timestamp": "2024-02-20T10:30:00Z",
    "presence": true
}
Example cURL
curl -X POST http://192.168.1.3:5000/api/vehicle/presence \
    -H "Content-Type: application/json" \
    -d '{
        "lane_id": "L1",
        "device_id": "D1",
        "timestamp": "2024-02-20T10:30:00Z",
        "presence": true
    }'
Expected Response
{
    "success": true,
    "message": "Vehicle presence recorded",
    "data": {
        "lane_id": "L1",
        "device_id": "D1",
        "timestamp": "2024-02-20T10:30:00Z",
        "presence": true
    }
}
Important Notes
  • All fields are required
  • Timestamp must be in ISO 8601 format
  • Lane ID must be a valid lane identifier
  • Device ID must be a valid device identifier
Vehicle ANPR API
Endpoint

POST http://192.168.1.3:5000/api/vehicle/anpr

Request Format
{
    "lane_id": "L1",
    "device_id": "D1",
    "timestamp": "2024-02-20T10:30:00Z",
    "image_url": "http://example.com/image.jpg",
    "plate_number": "ABC123",
    "confidence": 0.95
}
Example cURL
curl -X POST http://192.168.1.3:5000/api/vehicle/anpr \
    -H "Content-Type: application/json" \
    -d '{
        "lane_id": "L1",
        "device_id": "D1",
        "timestamp": "2024-02-20T10:30:00Z",
        "image_url": "http://example.com/image.jpg",
        "plate_number": "ABC123",
        "confidence": 0.95
    }'
Expected Response
{
    "success": true,
    "message": "ANPR data recorded",
    "data": {
        "lane_id": "L1",
        "device_id": "D1",
        "timestamp": "2024-02-20T10:30:00Z",
        "plate_number": "ABC123",
        "confidence": 0.95,
        "access_granted": true
    }
}
Important Notes
  • All fields are required
  • Timestamp must be in ISO 8601 format
  • Image URL must be accessible
  • Confidence must be between 0 and 1
Barrier Control API
Endpoint

POST http://192.168.1.3:5000/api/barrier/control

Request Format
{
    "lane_id": "L1",
    "device_id": "D1",
    "action": "open",
    "timestamp": "2024-02-20T10:30:00Z"
}
Example cURL
curl -X POST http://192.168.1.3:5000/api/barrier/control \
    -H "Content-Type: application/json" \
    -d '{
        "lane_id": "L1",
        "device_id": "D1",
        "action": "open",
        "timestamp": "2024-02-20T10:30:00Z"
    }'
Expected Response
{
    "success": true,
    "message": "Barrier command received",
    "data": {
        "lane_id": "L1",
        "device_id": "D1",
        "action": "open",
        "timestamp": "2024-02-20T10:30:00Z",
        "status": "executing"
    }
}
Important Notes
  • All fields are required
  • Action must be either "open" or "close"
  • Timestamp must be in ISO 8601 format
  • Device must be online and accessible
Health Check API
Endpoint

    "device_type": "barrier",
    "status": "online",
    "last_heartbeat": "2024-02-20T10:30:00Z",
    "error_message": null
}
Example cURL
curl -X POST http://192.168.1.3:5000/api/health/check \
    -H "Content-Type: application/json" \
    -d '{
        "lane_id": "L1",
        "device_id": "D1",
        "device_type": "barrier",
        "status": "online",
        "last_heartbeat": "2024-02-20T10:30:00Z",
        "error_message": null
    }'
Expected Response
{
    "success": true,
    "message": "Health check received",
    "data": {
        "lane_id": "L1",
        "device_id": "D1",
        "device_type": "barrier",
        "status": "online",
        "last_heartbeat": "2024-02-20T10:30:00Z",
        "error_message": null
    }
}
Important Notes
  • All fields are required
  • Device type must be one of: barrier, camera, sensor
  • Status must be one of: online, offline, error
  • Last heartbeat must be in ISO 8601 format
